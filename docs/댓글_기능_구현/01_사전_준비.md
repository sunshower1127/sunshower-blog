# Phase 1: 사전 준비 (GitHub 설정)

코드 구현 전에 GitHub에서 필요한 설정을 완료해야 합니다.

## 1단계: GitHub Discussions 활성화

### 작업
1. 블로그 GitHub 저장소로 이동
2. **Settings** 탭 클릭
3. **General** 섹션에서 **Features** 찾기
4. **Discussions** 체크박스 활성화
5. 저장

### 확인
- 저장소 상단 탭에 **Discussions** 탭이 나타남

---

## 2단계: giscus 앱 설치

### 작업
1. https://github.com/apps/giscus 방문
2. **Install** 버튼 클릭
3. 설치 옵션 선택:
   - **Only select repositories** 선택
   - 블로그 저장소 선택 (예: `username/sunshower-blog`)
4. **Install** 클릭하여 설치 완료

### 확인
- 저장소 Settings → Integrations → GitHub Apps에서 giscus 확인 가능

---

## 3단계: giscus 설정값 획득

### 작업
1. https://giscus.app/ko (한국어 버전) 방문

2. **저장소** 섹션:
   - 형식: `사용자명/저장소명` (예: `sunwoo-kim/sunshower-blog`)
   - 입력 후 체크 표시 확인 (저장소가 조건 충족하는지 검증)

   ⚠️ **주의**: 저장소가 **public**이어야 하고 Discussions가 활성화되어야 합니다.

3. **Discussion 카테고리** 섹션:
   - 추천: **"General"** 또는 새로 **"Comments"** 카테고리 생성
   - 발표(Announcements) 형식 선택

4. **페이지 ↔️ Discussions 매핑** 섹션:
   - 추천: **"pathname"** 선택
   - 각 블로그 포스트 URL 경로가 고유한 Discussion과 매핑됨

5. **Discussion 기능** 섹션:
   - ✅ **메인 포스트에 반응 남기기** 활성화 (👍❤️ 리액션)
   - ✅ **giscus를 게으르게 불러오기** 활성화 (성능 최적화)

6. **테마** 섹션:
   - **"Preferred color scheme"** 선택
   - 자동으로 브라우저 다크모드 설정 따름

---

## 4단계: 설정값 복사

페이지 하단 **giscus 사용** 섹션에 다음과 같은 스크립트가 생성됩니다:

```html
<script src="https://giscus.app/client.js"
        data-repo="사용자명/저장소명"
        data-repo-id="R_kgDOxxxxxx"
        data-category="General"
        data-category-id="DIC_kwDOxxxxxx"
        data-mapping="pathname"
        data-strict="0"
        data-reactions-enabled="1"
        data-emit-metadata="0"
        data-input-position="bottom"
        data-theme="preferred_color_scheme"
        data-lang="ko"
        data-loading="lazy"
        crossorigin="anonymous"
        async>
</script>
```

### 필수 복사 항목
다음 값들을 메모장에 복사해두세요:

| 항목 | 속성명 | 예시 |
|------|--------|------|
| 저장소 | `data-repo` | `sunwoo-kim/sunshower-blog` |
| 저장소 ID | `data-repo-id` | `R_kgDOxxxxxx` |
| 카테고리 | `data-category` | `General` 또는 `Comments` |
| 카테고리 ID | `data-category-id` | `DIC_kwDOxxxxxx` |

⚠️ **중요**: 이 값들은 다음 단계에서 설정 파일에 입력해야 합니다.
